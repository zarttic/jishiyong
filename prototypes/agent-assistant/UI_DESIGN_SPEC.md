# 及时用 UI 设计规范

此文档配合当前原型使用：

- 原型入口：`index.html`
- 应用图标：`app-icon.svg`

## 设计方向

最终方向为“保鲜墙 + 保鲜标签 + 小用便签”。

及时用不是企业库存系统，也不是聊天机器人。它的核心感受应该是：家里的东西被温柔地看管着。视觉上用保鲜墙、磁吸贴、保鲜标签、纸质折角和时间货架来建立记忆点。

## 应用图标

最终应用图标：`app-icon.svg`

当前最终选择：饮品符号版保鲜标签。

选择理由：

- 它直接表达“贴标签、管理到期时间”的产品核心。
- 饮品符号比食品、日用、药品更有“保鲜/冷藏”的即时联想，同时不会像药品一样显得过窄。
- 能覆盖食品、饮品、日用、药品、美妆等全品类。
- 和首页“保鲜墙”、新增页“贴标签流程”视觉语言一致。

交付文件：

- `app-icon.svg`：当前最终图标，使用饮品符号版。
- `app-icon-food.svg`：食品符号版。
- `app-icon-drink.svg`：饮品符号版，当前选定方向。
- `app-icon-daily.svg`：日用符号版。
- `app-icon-medicine.svg`：药品符号版。

图标元素：

- 圆角背景：冷雾浅绿到暖白，呼应生活工具和保鲜感。
- 保鲜墙背景：不使用对开门中缝，用冷雾面板、轻微高光和低对比横向分层线表示冰箱门/收纳柜。
- 主保鲜标签：白色纸面、右上折角、保鲜刻度。
- 主符号：饮品包装符号，放在原绿色方块位置，替代加号。
- 小用便签：暖黄色高对比浮层，从角落里抬出来，和青绿/暖白底色拉开色相差异；不做人形头像，只用两个眼点和轻微弧线表达助手感。

Android 落地建议：

- Adaptive icon background：使用 `#EFF6F2` 到 `#F8F5EC` 的冷雾浅色渐变。
- Adaptive icon foreground：保留主保鲜标签、暖黄色小用便签和低对比保鲜墙分层线。
- 小尺寸图标中不要保留过细文字，图标只靠形状识别。
- 导出时先测试 48dp、72dp、96dp 下的清晰度。

## 图标风格

图标走“包装符号”方向，不使用默认 Material 通用图标作为主视觉。

特征：

- 圆角线条。
- 线宽约 `1.7dp`。
- 少量淡填充，填充透明度约 12% 到 14%。
- 图标颜色来自分类色。
- 每个图标像标签上的印章。

建议先覆盖这些分类：

- 食品：碗/餐食
- 饮品：奶盒/瓶
- 日用：喷瓶
- 药品：胶囊
- 美妆：面膜/瓶
- 电子：手机
- 衣物：衣服
- 其他：收纳盒

## 首页

首页是“保鲜墙”，保留冰箱门的材质感，但不画真实对开门结构。

背景参考方向：

- 冷雾面板：当前基准方向。更轻、更干净，适合让标签和今日看板成为绝对主角。
- 磁吸保鲜墙：保留冰箱门的生活感，但取消中缝，用侧边光和低对比横线做空间暗示。
- 纸面贴板：偏手账和便签系统，亲和力强，但冰箱感会弱一些。
- 厨房浅砖：空间感最明确，生活场景强；需要注意别让网格显得太硬。
- 收纳层板：把物品分层放置的概念讲得最直接，适合强调时间货架。

结构：

1. 顶部品牌区
2. 今日保鲜看板
3. 小用便签
4. 搜索和分类贴
5. 时间货架
6. 底部小用语音入口

首页不放背景切换控件。背景方向属于个性化设置，避免把原型调试控件混入用户每天看的首页。

今日保鲜看板：

- 文案示例：`优先处理 3 件`
- 副文案：`牛奶今天到期，草莓和面膜也快到提醒线了。`
- 下方放保鲜热力层：稳定、临期、过期三段色带。

时间货架：

- `今天`
- `明天`
- `3 天内`
- `本周`
- `更晚`

每个分组用横向滚动保鲜标签，避免首页只是普通纵向列表。

## 保鲜标签

物品不再是普通卡片，而是一张纸质保鲜标签。

标签内容：

- 分类印章
- 剩余时间胶囊
- 物品名
- 位置/备注/数量
- 保鲜刻度
- 右上折角

材质状态：

- 新鲜：纸面干净，刻度完整，颜色饱和。
- 临期：纸面微暖，刻度变短。
- 今天到期：状态胶囊和边缘强调为暖红。
- 过期：降低饱和度，像旧标签。

## 小用助手

小用不是头像，不做固定麦克风按钮。

小用形态：

- 平时是一张有折角的小便签。
- 在首页贴到看板下方。
- 在分组旁表达“正在提醒这一组”。
- 语音时从底部入口展开成助手面板。
- 执行前必须展示结构化理解结果。

语气：

- 短、家常、克制。
- 不要说“主人”“智能管家”“开启高效生活”。

推荐文案：

- `我看过了，今天先处理牛奶最稳。`
- `我在听，可以直接说库存变化。`
- `我整理好了，确认后再改库存。`
- `这句没听清，没有改库存。`

## 小用底部面板

语音入口不是固定圆按钮，而是底部窄条：

```text
说一句库存操作
小用会先整理，再让你确认
```

展开面板包含：

- 小用便签头部
- 当前状态
- 语音波形
- 识别文本
- 结构化操作预览
- 确认和重说按钮

执行前预览示例：

```text
识别到   把牛奶用掉一瓶
动作     使用物品
结果     牛奶剩余 2 -> 1
```

## 新增页

新增页命名为：`贴一张保鲜标签`

保存按钮：`贴上标签`

交互结构：

- 顶部实时保鲜标签预览。
- 输入物品名称后标签文字变化。
- 选择分类后标签颜色和印章变化。
- 选择到期日后保鲜刻度变化。
- 小用便签提示提醒节奏。

这个页面应像“标签打印机”，而不是普通表单。

## 详情页

详情页命名为：`物品档案袋`

结构：

- 顶部档案袋形态。
- 大号物品标签。
- 剩余天数作为视觉焦点。
- 购买日期到到期日期的时间线。
- 信息行列表。
- 数量使用进度用小刻度，不用普通进度条。

数量示例：

```text
已用 1 / 2
[已用][未用]
```

## 少浪费报告

统计页命名为：`少浪费报告`

视觉像一张月度收据，而不是仪表盘。

结构：

- 月份
- 本月判断
- 浪费率
- 用掉 / 浪费 / 库存
- 分类占比条
- 小用建议

文案示例：

```text
本月情况稳定
饮品类临期较集中，下次少量多次补货。
```

## 小用巡视

这是一个特殊模式，不是聊天页。

用途：快速盘点临期物品。

结构：

```text
小用巡视
第 2 / 5 件

牛奶还在吗？
冷藏第二层 · 当前剩 2 件

[用掉了] [还在] [丢掉了]
```

交互目标：

- 带用户完成一次整理。
- 每次只问一件。
- 选择后进度条前进。

## 设置页

设置页命名为：`设置`

结构：

- 保鲜墙背景
- 语音配置状态

背景切换在设置中使用紧凑行项，不使用首页里那种大标签。每个选项包含一个小色块、背景名称和一句短说明。当前选中的背景显示右侧状态，不额外占用首页视觉空间。

当前默认背景选择：`冷雾面`

冷雾面落地要求：

- 不画对开门中缝。
- 不画抢眼的中部横线。
- 只使用轻微径向高光、顶部白雾和底部低对比暖色过渡。
- 标签和今日看板应该是第一视觉焦点。

语音配置：

- 设置页显示 `百度云语音` 的配置状态。
- 当前阶段要求发布包开箱即用，Release APK 必须带上百度云语音配置。
- 发布流程必须在缺少 `BAIDU_ASR_APP_ID`、`BAIDU_ASR_API_KEY`、`BAIDU_ASR_SECRET_KEY` 时失败，不能继续分发无语音兜底的包。

## 空状态

空状态不要画空箱子，直接展示第一张半透明标签。

文案：

```text
还没有贴上保鲜标签
从牛奶、药品、护肤品开始记录，到期前小用会提醒你。
```

按钮：

- `贴第一张标签`
- `对小用说一句`

## 动效规范

动效是这套设计的重要部分，但必须克制。

原则：

- 只动画 `transform` 和 `opacity`。
- 不动画 `width`、`height`、`top`、`left`。
- 动效时长多数控制在 220ms 到 760ms。
- 使用柔和回弹，不使用夸张弹跳。
- 支持系统减少动态效果设置。

当前原型包含：

- 页面切换：冰箱门上扫过一层短暂巡视光。
- 首页进入：看板、分类、时间货架分层进入。
- 标签进入：像贴纸贴上去，有轻微旋转和回弹。
- 热力层：绿、黄、红分段顺序铺开。
- 新增页：标签像从打印机吐出。
- 详情页：档案袋打开。
- 报告页：收据落下，条形逐条填充。
- 小用入口：轻微呼吸。
- 小用面板：底部入口收起，面板内容分步进入。
- 巡视模式：选择按钮后确认，进度条前进。

Compose 落地建议：

- 页面切换：`AnimatedContent` 或 `AnimatedVisibility`。
- 标签进入：`scale`、`alpha`、`rotationZ`。
- 小用面板：`ModalBottomSheet`。
- 波形：Canvas 或一组高度变化的圆角条。
- 热力层：Canvas 或 Row 权重分段。
- 减少动态：跟随系统 animation scale，或提供应用内关闭动效开关。

## Compose 组件拆分建议

建议组件：

- `FreshnessWallScreen`
- `TodayFreshnessBoard`
- `FreshnessHeatBar`
- `XiaoyongNote`
- `XiaoyongBottomSheet`
- `FreshnessLabelCard`
- `TimeShelfSection`
- `CategoryStamp`
- `LabelPrinterForm`
- `ItemFolderDetail`
- `WasteReceiptReport`
- `XiaoyongInspectionFlow`

不要一次性大重构数据层。第一阶段只替换 UI 组件，并复用现有 ViewModel 状态。

## 开发顺序建议

1. 建立新主题色和基础 Shape。
2. 实现 `FreshnessLabelCard` 与 `CategoryStamp`。
3. 改首页为保鲜墙和时间货架。
4. 替换语音弹窗为小用底部面板。
5. 改新增页为标签打印机流程。
6. 改详情页为档案袋。
7. 改统计页为少浪费报告。
8. 最后增加小用巡视模式。

## 不做的事

- 不做人形、动物或卡通助手。
- 不使用科幻 AI 紫蓝渐变。
- 不把小用做成固定圆形悬浮按钮。
- 不把首页做成普通 3 卡片指标面板。
- 不用过度弹跳或大面积动画。
